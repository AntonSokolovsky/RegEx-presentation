Regular Expressions, RegEx

What Are Regular Expressions?
  • A regular expression is a pattern that the regular expression engine attempts to match in input text.
  • The concept of regular expressions began in the 1950s, when the American mathematician Stephen Cole Kleene formalized the concept of a regular language
  • Regular expressions are used in search engines, in search and replace dialogs of word processors and text editors, in text processing utilities such as sed and AWK, and in lexical analysis.
  • Regular expressions are supported in many programming languages.
Basic Syntax and Components
  • Character Classes: "\d" and "\w"
  • Quantifiers:"*", "+", and "?".
  • Anchors: "^" and "$"
  • Alternation and Grouping: "|" ,"()" and "[]"
Practical Implementation
Validating Email Addresses: A regex pattern to validate email addresses. 
let trueEmail = 'example@email.com';
let falseEmail = 'example&email';

let regexp = /[\w\.-]+@[\w\.-]+\.\w{2,4}/gi;

console.log(regexp.test(trueEmail)); //true
console.log(regexp.test(falseEmail)); //false

Extracting Information from Log Files: Regex to parse timestamps, log levels, and log messages. 
const logData = `
	2023-08-01 12:15:32 192.168.1.1 GET /api/data
	2023-08-01 12:20:45 192.168.1.2 POST /api/user
	2023-08-01 12:25:01 192.168.1.3 GET /home
`;
const logPattern = /(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) (\d+\.\d+\.\d+\.\d+) (\w+) (.+)/g;
let match;
while ((match = logPattern.exec(logData))) {
	const timestamp = match[1];
	const ipAddress = match[2];
	const httpMethod = match[3];
	const requestUrl = match[4];
	console.log(`Timestamp: ${timestamp}, IP Address: ${ipAddress}, Method: ${httpMethod}, URL: ${requestUrl}`);
  }
//Timestamp: 2023-08-01 12:15:32, IP Address: 192.168.1.1, Method: GET, URL: /api/data
//Timestamp: 2023-08-01 12:20:45, IP Address: 192.168.1.2, Method: POST, URL: /api/user
//Timestamp: 2023-08-01 12:25:01, IP Address: 192.168.1.3, Method: GET, URL: /home
Data Validation and Sanitization: Using regex to validate user input (e.g., phone numbers). 
function validatePhoneNumber(phoneNumber) {
// Define the regular expression pattern using RegExp object
const pattern = new RegExp("^\\+[1-9]{1}[0-9]{0,2}-[2-9]{1}[0-9]{2}-[2-9]{1}[0-9]{2}-[0-9]{4}$");
// Test the phone number against the pattern
if (pattern.test(phoneNumber)) {
	console.log("Phone number is valid");
	return true;
} else {
	console.log("Phone number is not valid");
	return false;
}
}

validatePhoneNumber("+123-456-789-1234");  // true
validatePhoneNumber("+0-123-456-7890");    // false
validatePhoneNumber("+12-012-123-1234");   // false

Best Practices and Tips

  • Test and iterate step-by-step to avoid unexpected matches.
  • Leverage online regex testers and cheat sheets for quick reference.
